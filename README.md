## Реализованы классы

### 1. **Casino (casino.py)**

Основной класс, управляющий всем казино:

- Хранит всех игроков и гусей
- Управляет случайными событиями

**методы:**
- `add_player(player)` - добавление игрока в казино
- `add_goose(goose)` - добавление гуся в казино
- `perform_random_event()` - выполнение случайного события
- `event_goose_steal_chips()` - событие кражи фишек гусем
- `event_goose_steal_money()` - событие кражи фишек гусем
- `event_player_bet()` - событие ставки в рулетке
- `random_player()` - рандомный выбор игрока
- `random_goose()` - рандомный выбор гуся
- `event_honk_goose()` - событие крика гуся-крикуна
- `event_play_goose()` - событие ставки в рулетке гусем-игроком
- `event_player_buy_chips()` - cобытие покупки фишек в казино.


### 2. **Player (player.py)**

Класс игрока с характеристиками:

- Имя и текущий баланс в долларах
- Коллекция фишек разных номиналов (1$, 5$, 25$, 100$)
- Уровень паники (0-100)

**методы:**
- `buy_chips()` - покупка фишек за деньги
- `rulette()` - игра в рулетку
- `transfer_money()` - обмен фишек на деньги

### 3. **Классы гусей (goose.py)**

#### **Goose (обычный гусь)**

- Базовая реализация гуся
- Может воровать фишки и деньги у игроков
- Вызывает панику у игроков

#### **HonkGoose (гусь-крикун)**

- Наследуется от Goose
- Может кричать на игроков, увеличивая свою способность к краже
- Имеет усиленную кражу после крика

#### **PlayGoose (гусь-игрок)**

- Может играть в рулетку
- Ворует только фишки (не интересуется деньгами)
- Хранит свои фишки как игрок

### 4. **ChipCollection (chip_collection.py)**

Коллекция для управления фишками:

- Хранение фишек 4-х номиналов (1$, 5$, 25$, 100$)
- Операции сложения, вычитания, умножения
- Проверка наличия фишек

**Реализованные операторы:**
- `+=` - добавление фишек
- `-=` - удаление фишек
- `*=` - умножение фишек
- `>=` - сравнение коллекций

### 5. **UsersCazino (users.py)**

Коллекция для хранения пользователей казино:

- Реализует интерфейс, похожий на словарь
- Поддерживает итерацию, индексацию, получение длины, удаление через del

## Особенности реализации

### Система паники

- Гуси вызывают панику у игроков при краже
- При достижении 100 паники игрок сбегает из казино (удаляюься из общего списка)
- Гуси-крикуны имеют увеличенную силу паники

### Механика кражи

- Гуси могут воровать ограниченное количество фишек за раз
- Гуси-крикуны могут усиливать свою кражу после крика
- Гуси-игроки воруют только фишки

### Игровая механика

- Рулетка имеет 10 секций с разными множителями (3x, 2x, 0x)
- Игроки могут покупать фишки разных номиналов
